# 05 Editorial Rules And Execution

## Editorial promise
The site explains the likely next action after a buried-tank trigger. It does not pretend to diagnose the property from a webpage.

## Voice rules
- calm
- specific
- transaction-aware
- scenario-first
- never sensational

## Mandatory language rules
Use:

- `may`
- `often`
- `likely`
- `verify`
- `in this state`
- `based on the available record or guidance`

Avoid:

- `definitely`
- `always`
- `you must remove it`
- `you are safe if`
- `this proves there is no tank`

## Claims the site can make
- a state may treat residential heating oil tanks through a particular process
- a missing record is not the same as proof that no tank exists
- a sweep or locate may be the honest first step before removal decisions
- some cases escalate from closure into reporting or cleanup workflow

## Claims the site must not make
- tank presence certainty without evidence
- contamination certainty without proper investigation
- legal certainty on buyer-seller liability in every case
- exact cost promises

## Page-family rules

### State hub
- must explain the strongest trigger states in that state
- must route users to the correct next page quickly
- must show a short official source stack

### Buyer-seller page
- must prioritize verification and records before dramatic price talk
- must explain what documents matter
- must separate `suspected`, `confirmed`, and `leak concern`

### Sweep page
- must explain when sweep first is more honest than quote first
- must avoid pretending every user needs removal immediately

### Remove-versus-abandon page
- must state that the answer is state-specific
- must explain what evidence changes the decision

### Leak-and-cleanup page
- must state that cleanup and reporting can be separate from simple removal
- must be extra careful not to overstate contamination

### Cost page
- must stay directional
- must explain scenario assumptions
- must link back to verification routes

## Execution order
1. Home and trust pages
2. Four public state hubs plus reserve-state drafts
3. Four public state buyer-seller pages
4. Four public state sweep pages
5. Four public state records pages
6. Guide pages
7. Remove-versus-abandon and leak routes

## Definition of done for a public page
- has a clear trigger state
- has a state-aware answer
- has a source stack
- has one primary CTA
- has a `what not to assume` section
- has at least one internal link to the next likely route

## Editorial kill rule
If a draft page reads like a general homeowner article rather than a triggered decision page, stop and rewrite the angle.
