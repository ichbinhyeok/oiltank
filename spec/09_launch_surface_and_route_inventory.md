# 09 Launch Surface And Route Inventory

## Launch rule
The first public version should feel narrow and sharp, not broad and half-true.

## Launch cohort
- `new-jersey`
- `new-york`
- `connecticut`
- `maine`

## Reserve state
- `massachusetts`

## Core route families per state
- overview
- buyer-seller
- sweep-and-locate
- records-and-proof
- removal-vs-abandonment
- leak-and-cleanup

## Phase 1 index rule
Only these route families should index in the first release:

- overview
- buyer-seller
- sweep-and-locate
- records-and-proof

`removal-vs-abandonment`, `leak-and-cleanup`, and `cost-direction` should exist only as support inventory or later launch work until the first wedge proves traction.

State hubs should foreground only `buyer-seller`, `sweep-and-locate`, and `records-and-proof`. Advanced support routes should stay contextual rather than acting as front-door navigation.

## Why this launch surface is enough
- captures the strongest trigger states
- keeps the first release centered on home-sale risk
- gives enough pages to test internal linking
- supports multiple commercial paths without bloating the surface
