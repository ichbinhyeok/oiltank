# Route Promotion Board

## Purpose
This board exists so future agents do not rely on founder memory.

When metrics exist, the agent should review this board first and then tell the user whether any route deserves promotion.

## Current status
No metrics yet.

Default state for all held route families is `hold`.

## Held route families

### Removal-versus-abandonment
- Current status: `hold`
- Current surface: support or `noindex`
- Promote when:
  - indexed buyer-seller, sweep, or records pages show repeated confirmed-tank confusion
  - source freshness is green
  - state-specific logic is strong enough to beat a generic explainer
- Do not promote when:
  - the only signal is broad removal curiosity

### Leak-and-cleanup
- Current status: `hold`
- Current surface: support or `noindex`
- Promote when:
  - admin leads or contact notes show repeated leak concern
  - the state has clear reporting and cleanup guidance
  - the route can separate suspicion from contamination workflow safely
- Do not promote when:
  - the evidence is just fear-driven browsing

### Cost-direction
- Current status: `hold`
- Current surface: support or `noindex`
- Promote when:
  - indexed routes already show click and scenario evidence
  - cost guidance is source-backed
  - the route stays decision-first
- Do not promote when:
  - it would become a thin price page

### Market overlays
- Current status: `hold`
- Current surface: not built or `noindex`
- Promote when:
  - the parent state route has demand
  - the market materially changes the answer
  - maintenance burden is justified by commercial density

### Partner finder pages
- Current status: `hold`
- Current surface: not built or `noindex`
- Promote when:
  - the route family already has demand
  - partner coverage exists
  - the page adds routing value rather than becoming a thin directory
